Our project was to create chrome extensions, written in JavaScript, to better help detect phishing, the practice of sending fraudulent emails to procure sensitive information. One of the biggest problems with current phishing detectors was their speed and CPU usage, as these detectors frequently crosschecked between server-side copies of a webpage and the actual webpage to detect discrepancies. Our project was to create lightweight phishing detectors that did not compromise accuracy, by identifying scripts, frontend changes, and common URL patterns commonly used in phishing schemes, and then highlighting possible phishing schemes for the user to see. After writing the paper using LaTeX, it was presented in Dallas at the ACM international conference. My biggest project that has probably incorporated all my skills has been creating my software for my company, Linkpedia (which you can read more about above or at linkpedia.co). The software included a few main parts- user authentication and profile management, data visualization, and adding content and links. To create a user authentication, I originally used Django’s default system, but then later transitioned to creating my own. There are two types of users that could use our software- normal users, anybody, essentially, and experts. A big hurdle I had to overcome was figuring out how to validate that a person signing up, was in fact, an expert. Eventually, I overcame this by adding an expert rating system that could be honed by a community of experts. This system includes a LinkedIn API authentication, scraping the experts entered accomplishments and measuring them against common google results and acquiring recommendations from already proven experts. Data visualization was probably the most challenging part of creating the software- the core aspect of it was to take all the links and content that experts and users were adding, and display it geospatially on a google maps interface. Since the google maps API is written for JavaScript exclusively, I had to use a python library to output a JSON file using data from the MySQL database, and then store that JSON file locally and use it to generate the visualization. Adding content and links was straightforward- I had to learn how to create stylish forms, clean those forms and add error messages when something went wrong, and then add data to MySQL. Generating tags was also a key part of the project, to help suggest links that could be made. I did this with the help of the datamuse API, for which you need to file a GET request with your query and then receive a JSON file output that could easily be read by a python library. Some additional parts to the software include generating an efficient search algorithm, ranking links and contents based on crowdsourced input, and simulating a browser environment in which users can view external URLs.
